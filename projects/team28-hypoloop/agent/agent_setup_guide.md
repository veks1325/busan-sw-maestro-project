# Autonomous ML Agent Architecture Guide

이 문서는 LangGraph를 활용하여 LLM(에이전트)이 수백 개의 머신러닝 실험을 **알아서 코드를 작성하고, 실행하고, 평가하도록** 구성하기 위한 아키텍처 및 설정 가이드입니다.

## 1. 프롬프트 및 컨텍스트 주입 (System Prompt & Context)
에이전트가 코드를 작성할 때, 프로젝트의 데이터 로드 방식이나 환경(경로 등)을 정확히 인지하도록 가이드라인을 제공해야 합니다.
- **역할 부여:** "너는 데이터 분석 및 머신러닝 모델링을 수행하는 자율형 AI 엔지니어다."
- **템플릿 제공:** 에이전트가 코드를 짤 때 참고할 수 있는 모범 답안지(Few-shot Example)인 `exp/sql_to_pandas_template.py`의 구조를 시스템 프롬프트에 주입하거나, 에이전트가 시작 시 도구를 이용해 해당 파일을 읽어보도록 지시합니다.

## 2. 도구 (Tools) 연결
에이전트가 스스로 물리적인 작업(파일 작성, 코드 실행)을 수행할 수 있도록 파이썬 도구를 연결합니다. (`tool.py` 활용)
- **`write_file(path, content)`**: 에이전트가 구상한 ML 파이프라인 코드(예: `exp/rf_experiment.py`)를 실제 파이썬 스크립트로 저장합니다.
- **`execute_command(command)`**: 작성한 코드를 터미널에서 실행(`python exp/rf_experiment.py`)하고, 결과(Stdout) 및 에러(Stderr)를 에이전트에게 반환합니다.
- **`read_file(path)`**: 실행 결과 로그, 기존 코드, 데이터 스키마 등을 읽어 들여 분석합니다.

## 3. LangGraph 상태(State) 및 순환 루프 구성
에이전트는 일회성 문답이 아니라 지속적인 피드백 루프(Agentic Workflow)를 돌면서 코드를 완성합니다. 이를 위해 LangGraph의 `StateGraph`를 다음과 같이 구성합니다.

1. **[계획 - Plan]**
   에이전트가 실험 방향을 잡기 위해 종합적인 **'Goal 문서(목표 정의서)'**를 먼저 구성하고 이를 바탕으로 계획을 세웁니다. Goal 문서에는 다음 항목들이 포함됩니다:
   - **이전 실험 기록 (MLflow):** 과거에 어떤 파라미터로 학습했고 성능이 어땠는지 확인하여 시행착오를 줄입니다. (첫 실험이라 기록이 없으면 비워둡니다.)
   - **이전 피처들 (Feast):** 기존에 어떤 파생 변수나 피처들이 생성되어 있는지 파악하고 재사용 또는 신규 생성 여부를 결정합니다.
   - **데이터 카드 (DataCard):** 타깃 컬럼, 각 변수의 의미, 도메인 지식 등 유저가 UI에서 입력한 핵심 정보.
   - **태스크 (Task):** 분류(Classification)인지 회귀(Regression)인지 등 머신러닝의 근본적인 목적.
   - **산식 및 지표 (Metric):** 에이전트가 모델 성능을 평가할 명확한 기준 산식 (예: Accuracy, RMSE, F1-score 등).
2. **[학습 코드 생성 - Generate Training Code]**
   - 1번 [계획] 단계의 최종 산출물인 **'가설(Hypothesis)'**을 검증할 모델 학습 코드를 작성합니다. (이 가설은 에이전트가 스스로 세우거나, 유저가 직접 입력할 수 있습니다.)
   - 작성된 학습 스크립트에는 가설을 테스트할 **피처 엔지니어링 로직**과 실험 결과를 기록할 **MLflow 추적(Tracking) 코드**가 반드시 포함되어야 합니다.

3. **[EDA 및 평가 코드 생성 - Generate EDA/Evaluation Code]**
   - 학습이 성공적으로 완료되면, 그 결과를 분석하고 자산화하는 두 번째 스크립트를 작성합니다.
   - **피처 스토어 저장:** 학습 시 생성된 유용한 파생 피처들을 Feast 피처 스토어에 공식 등록하고 저장하는 코드를 포함합니다.
   - **시각화 코드 생성:** 피처 중요도(Feature Importance) 등을 시각화하여 이미지 파일(예: `.png`)로 저장하고 핵심 수치 결과를 출력(Stdout)하도록 작성합니다.
   - **에이전트 주도 리포트 작성:** 스크립트가 직접 문서를 만드는 대신, 에이전트가 출력된 지표와 시각화 결과를 바탕으로 `write_file` 도구를 직접 사용하여 어떤 피처가 효율/비효율적이었는지 상세히 설명하는 마크다운(`.md`) 리포트를 작성합니다.

4. **[실행 - Execute]**
   - `execute_command` 도구를 사용해 위에서 작성한 스크립트들을 차례대로 실행합니다.

5. **[피드백 및 수정 - Feedback & Self-Correction]**
   - **에러 발생 시:** 에러 로그를 읽고 이전 단계로 돌아가 스스로 디버깅하여 코드를 수정합니다.
   - **정상 작동 시:** 출력된 성능 지표를 평가하고, 리포트를 바탕으로 더 나은 가설을 세워 다음 실험 루프를 준비합니다.

## 4. 안정성 장치 (Safety & Human-in-the-loop)
에이전트가 끝없이 코드를 짜고 실행하며 리소스를 낭비하지 않도록 제어 장치를 추가합니다.
- **최대 루프 횟수 제한:** LangGraph의 `recursion_limit`을 설정하여 에이전트가 최대 5번~10번까지만 재시도하도록 제한합니다.
- **사용자 승인 (Human-in-the-loop):** 코드를 터미널에서 실행하기 전에 (또는 최초 1회 실행 전에) 사용자에게 코드를 보여주고 승인(`continue` 또는 `reject`)을 받도록 LangGraph의 `interrupt_before` 기능을 활용합니다.
